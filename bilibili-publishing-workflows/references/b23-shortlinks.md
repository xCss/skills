# Bilibili `b23.tv` Shortlink Notes

Session-derived notes for Bilibili poster/link generation.

## Key behavior

- Official-looking shortlinks such as `https://b23.tv/aM8Jyok` are server-side shortlink mappings generated by Bilibili share flows.
- The token after `b23.tv/` (e.g. `aM8Jyok`) is **not** a local encoding/hash of the BV ID and cannot be deterministically calculated from `BVxxxx` alone.
- Resolving a real shortlink returns a 302 to a canonical video URL with tracking params. Example observed:

```text
https://b23.tv/aM8Jyok
→ https://www.bilibili.com/video/BV1Ai9eBKEU4?...&unique_k=aM8Jyok&...
```

Typical redirect params may include `unique_k`, `share_source`, `share_medium`, `share_session_id`, `timestamp`, `up_id`, etc.

## What was tested

- `POST https://api.bilibili.com/x/web-interface/share/add` with `aid` can return OK or duplicate-share responses, but it increments/records share count and does **not** return a `b23.tv/xxxxxx` short URL.
  - Example success shape: `{"code":0,"message":"OK","ttl":1,"data":...}`
  - Example duplicate shape: `{"code":71000,"message":"重复分享","ttl":1}`
- `POST https://api.bilibili.com/x/share/click` and guessed shortlink endpoints generally returned request errors/404/405 during testing.
- No stable public endpoint was found that accepts a BV ID and returns an official `b23.tv/<unique_k>` shortlink without app share context/login/anti-abuse controls.

## Poster generation rule

For Bilibili posters:

1. If the user's original input contains a real `b23.tv/<token>` link, preserve and display that exact shortlink; generate the QR for that shortlink if appropriate.
2. If the user only provides a BV/AV/canonical Bilibili URL, using `https://b23.tv/BVxxxx` is acceptable as a compact redirect URL when verified: `b23.tv/BVxxxx` currently returns 302 to `https://www.bilibili.com/video/BVxxxx`. Make clear this is a redirect form, not an official `unique_k` shortlink token.
3. Use one of these forms:
   - `https://b23.tv/BVxxxx` as the compact poster/QR display link after verifying redirect behavior
   - `https://www.bilibili.com/video/BVxxxx/` as the canonical fallback
   - display `BVxxxx` only when space is extremely constrained
4. If a future authenticated/app-share API is discovered, document required cookies/signing/user-agent parameters here before using it.
